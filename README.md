# Hospital-Management-System-MongoDB


 📌 Project Overview

This project is a Hospital Management System developed using MongoDB (NoSQL database) to efficiently manage patient, doctor, and appointment data.

 🎯 Objectives

* Manage large amounts of hospital data efficiently
* Reduce manual paperwork and errors
* Provide quick access to patient records
* Perform data analysis using aggregation

 ⚙️ Tech Stack

* Frontend: HTML, CSS, JavaScript
* Database: MongoDB
* Query Language: MongoDB Query Language (MQL)
* Tool: Studio 3T

 Data Model

The system consists of three main collections:

* Patients
* Doctors
* Appointments

🔧 Features

* CRUD Operations (Create, Read, Update, Delete)
* Embedded Documents for patient history
* Aggregation Queries (doctor-wise analysis)
* Indexing for faster performance

📊 Key Concepts Used

* NoSQL Database
* Data Modeling
* Aggregation Pipeline
* Indexing
